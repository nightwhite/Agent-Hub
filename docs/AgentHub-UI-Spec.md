# Agent Hub UI 规范（v2）

## 目标
- 与 DevBox 同产品语义保持一致：克制、紧凑、系统工具感。
- 默认面向 Sealos 窗口模式，不以全屏网页为设计基准。
- 统一尺寸与层级，禁止“页面各写各的”视觉漂移。

## 适用范围
- `web/src/app/pages/agent-hub/**`
- `web/src/components/ui/**`
- `web/src/components/business/**`（仅 Agent Hub 使用区域）

## 信息架构（MVP）
- 实例列表页：看状态、搜索、进入详情、创建实例。
- 模板市场页：筛选并选择模板，进入创建页。
- 创建页：填参数并提交创建。
- 实例详情页：概览 / 文件 / 设置 / 对话 / Web UI（按模板能力显隐）。

## 基础原则
- 单一基线：交互控件默认高度统一为 `36px`（Tailwind: `h-9`）。
- 密度优先：正文默认 `12px`，说明文案 `11px`，标题按层级上浮，不做大号营销化标题。
- 一页一滚动主轴：每个工作区只允许一个主要纵向滚动容器。
- 左右结构固定：详情页必须保留 `Sidebar + Workspace`。
- 组件优先：页面禁止直接手写“另一套按钮/输入框样式”，必须复用 `ui` 组件。

## 尺寸与排版 Token（执行级）

### 字号层级
- 页面主标题：`text-[1.28rem]/7`（约 20px）
- 区块标题：`text-[1.02rem]/6`（约 16px）
- 组件标题：`text-[13px]/6` 或 `text-[12px]/5`（按密度选一）
- 正文：`text-[12px]/5`
- 辅助说明/Hint：`text-[11px]/5`
- 状态胶囊：`text-[10px]/4`

### 控件高度
- 按钮：`h-9`（默认）
- 紧凑次级按钮：`h-8`（卡片内动作）
- 输入框/选择框/搜索框：`h-9`
- 禁止在同一工具栏内混用 `h-8 / h-9 / h-10` 三档。

### 圆角与边框
- 卡片：`rounded-xl` 或 `rounded-2xl`（二选一，不混杂）
- 控件：`rounded-lg`
- 边框：`border-[0.5px] border-zinc-200`
- 禁止使用左侧强调条（`border-left > 1px`）作为状态表达。

### 间距基线（4pt）
- 主要内容区间距：`gap-2.5` / `gap-3`
- 卡片内上下间距：`py-2.5` ~ `py-3.5`
- 标签与字段间距：`space-y-1.5`
- 禁止大量使用 `mt-6+` 造成窗口模式挤压。

## 组件规范

### Button（`web/src/components/ui/Button.tsx`）
- 统一 API：
  - `variant`: `primary | secondary | ghost | danger`
  - `size`: `md | sm | xs`（`md` 默认）
- 复用规则：
  - 列表页、头部、主操作用 `size="md"`
  - 卡片内部次要操作用 `size="sm"`
  - 超密集行内按钮（如文件行内“进入/编辑”）用 `size="xs"`
  - 禁止同工具栏混用 `sm/md`（除非有明确主次关系）
- 禁止项：
  - 禁止在业务页面用 `className` 改出第三套按钮尺寸
  - 禁止单独把“创建 Agent”做成更高按钮

### 输入类控件
- `Input` / `Select` / `SearchField` 统一 API：
  - `size`: `md | sm`（`md` 默认）
- 默认：`h-9` 与 `12px` 文本。
- Label：`12px`；Hint：`11px`。
- 只在空间极紧（卡片嵌套动作区）可用 `h-8`。

### Size 对照表（强制）
- `xs`: `h-7 px-2 text-[10px]`（仅超密集行内按钮）
- `sm`: `h-8 px-2.5 text-[11px]`（卡片内次级动作）
- `md`: `h-9 px-3 text-[12px]`（页面默认）

### Header
- 列表页头、详情页头统一最小高度 `72px` 左右（`min-h-[72px]`）。
- 主标题控制在 `~20px`，不使用 `text-2xl` 及以上。
- 右侧动作区优先保持单排；空间不足时可换行，但尺寸保持一致。

### Sidebar
- 详情左侧导航宽度：`56px`（窄）到 `78px`（宽屏）。
- 图标推荐 `18px`，文案 `10~11px`。
- 允许侧栏内部滚动，防止低高度遮挡。

## 页面规范

### Agents List
- 头部搜索、浏览模板、创建 Agent 使用统一控件高度 `h-9`。
- 表格区保持高密度，避免过厚行高和过大按钮。

### Agent Detail - Overview
- 双列断点提前（约 `1120px`），避免中间宽度拥堵。
- 信息卡文字层级固定：标题 16、正文 12、辅助 11、tag 10。

### Agent Detail - Files
- 顶部工具栏、中栏动作、右栏操作三处按钮统一密度。
- 空态/加载态最小高度随窗口收缩，避免“半屏只剩空白”。
- Markdown 预览与编辑双栏仅在宽度允许时启用。

### Agent Detail - Settings
- 字段输入区优先网格化，避免超窄宽度横向挤压。
- Runtime/Agent 两块卡片间距一致，保存按钮统一 `h-8~h-9`。

## 响应式规则
- 宽度不足时优先：减少横向间距 > 组件换行 > 降低次要信息展示。
- 高度不足时优先：保证主内容可滚动 > 固定顶部操作 > 减少空态占高。
- 不允许出现水平滚动条作为常规布局手段。

## 禁止项
- 禁止同页存在多套按钮尺寸基线（例如 `h-8 + h-10` 混用）。
- 禁止营销化大字标题和过重阴影。
- 禁止没有统一 token 的“临时 className 放大”。
- 禁止以兜底逻辑掩盖 UI 结构问题（如靠强制 min-width 撑布局）。

## 组件复用标准（强制）
- 只允许从 `web/src/components/ui` 导入基础控件，不允许在业务目录复制一个“Button2/Input2”。
- 新页面默认先用现有组件组合；只有现有 API 不能满足时，先扩展 UI 组件 API，再改页面。
- 所有新控件必须满足：`variant + size + disabled + error` 四项可控（适用项）。

## 当前标准组件清单
- `Button`: 动作触发
- `Input`: 文本输入
- `Select`: 选项输入
- `SearchField`: 搜索输入
- `Modal`: 浮层容器
- `StatusBadge`: 状态展示
- `EmptyState`: 空态展示

## 页面实现约束（给开发和 AI）
- 页面里遇到以下样式需求，优先使用组件参数，不要写临时样式：
  - “按钮小一点” -> `size="sm"`
  - “按钮主次区分” -> `variant`
  - “搜索框和按钮等高” -> 同时使用 `size="md"`
  - “卡片里紧凑输入” -> `Input/Select size="sm"`
- 如果必须用 `className` 覆盖，只允许做布局（宽度、对齐、间距），不允许改尺寸基线。

## 落地顺序
1. 先改基础组件基线（Button/Input/Select/SearchField）。
2. 再改列表页与详情页头部。
3. 再改三个工作区（Overview/Files/Settings）。
4. 最后做窗口模式回归检查（窄宽 + 低高）。
